# /register-scale スキル

## 概要

`papers/<尺度id>/` ディレクトリに置いた論文PDF群から尺度データを抽出し、アプリへの登録まで自動で行う。
論文本文・項目リスト・補足資料など複数のPDFをまとめて読み込める。

## 呼び出し形式

```
/register-scale papers/<尺度id>/
```

引数なしで呼んだ場合は `papers/` 内のサブディレクトリ一覧を表示して選択を促す。

---

## ディレクトリ構造

```
papers/
└── <尺度id>/               # 尺度ごとにディレクトリを作成
    ├── paper.pdf           # 論文本文（規準値・構成・APA情報など）
    ├── items.pdf           # 質問項目一覧（別紙になっている場合）
    └── *.pdf               # その他補足資料（付録・追加研究など）
```

ファイル名は自由。ディレクトリ内のPDFをすべて読み込む。

---

## 実行手順

### Step 1: ディレクトリとPDFの確認

指定されたディレクトリ内のファイルを確認する。

```
Bash(ls papers/<尺度id>/)
```

ディレクトリが存在しない場合は `papers/` 内のサブディレクトリ一覧を表示してユーザーに確認する。

### Step 2: 全PDFを読み込む

ディレクトリ内のPDFをすべて読み込む。ファイルが多い場合は内容の概要をまず確認してから詳細を読む。

```
Read(papers/<尺度id>/paper.pdf)
Read(papers/<尺度id>/items.pdf)
# ... ディレクトリ内の全PDFを読む
```

各ファイルの役割（本文・項目リスト・補足など）を把握してから情報統合する。

### Step 3: 既存登録の確認

重複登録を防ぐため、現在の登録済み尺度を確認する。

```
Read(lib/scales/index.ts)
```

### Step 4: 全ファイルから尺度情報を抽出・統合

複数PDFの内容を統合して以下の情報を読み取る。抽出が曖昧な箇所はユーザーに確認する。

**必須情報：**
- 尺度の正式名称・略称
- 全質問文と項目番号（項目リストが別PDFの場合はそちらを優先）
- 逆転項目の一覧
- 選択肢の文言とスコア値（何件法か）
- 下位尺度の構成（どの項目がどの下位尺度に属するか）
- APA形式の引用文献情報（著者・年・タイトル・雑誌・DOI）

**あれば取得する情報：**
- 規準値（M, SD, N, 対象者）— **項目平均ベースか素点合計ベースかを確認**
- 教示文
- クラスタ分析の結果（クラスタ名・判別条件）
- 対象者（大学生・成人一般など）

**複数ファイルがある場合の優先順位：**
- 質問文 → 項目リストPDFを優先、なければ論文付録
- 規準値・構成 → 論文本文
- 矛盾がある場合 → ユーザーに確認

### Step 5: norm_mean の単位確認

規準値（M/SD）が見つかった場合、**項目平均か素点合計かをPDFの記載から判断**する。

- 素点合計ベースの場合：`norm_mean = M ÷ 項目数`、`norm_sd = SD ÷ 項目数` に変換する
- 判断できない場合：ユーザーに確認してから入力する

### Step 6: IDの決定

`meta.id` を決定する。ルール：
- `papers/` のサブディレクトリ名をそのまま使う（kebab-case）
- 既存IDと重複しないこと
- URLとして使われるため、英数字とハイフンのみ

候補をユーザーに提示して確認を取る。

### Step 7: JSONファイルの生成

`lib/scales/<id>.json` を作成する。スキーマは `lib/types.ts` の `Scale` 型に従う。
テンプレートは `docs/scale-registration-guide.md` を参照。

生成したJSONをユーザーに提示して**確認を取ってから**ファイルを書き込む。

### Step 8: レジストリへの登録

`lib/scales/index.ts` を編集してimportと配列への追加を行う。

```typescript
import newScale from "./<id>.json";

const scaleList: Scale[] = [
  // 既存 ...
  newScale as Scale,
];
```

### Step 9: 型チェック

```bash
Bash(cd /home/shun/dev/psych-scales && npx tsc --noEmit)
```

エラーがあれば修正する。

### Step 10: 動作確認の案内

登録完了後、ユーザーに以下を案内する：
1. `npm run dev` でdevサーバーを起動してホーム画面を確認
2. 尺度をクリックして回答・結果画面を確認
3. 問題なければコミット

### Step 11: コミット

ユーザーから問題ないとの確認が取れたら：

```bash
git add lib/scales/<id>.json lib/scales/index.ts
git commit -m "feat: <尺度名>を登録"
```

---

## 注意事項

- 著作権の観点から、論文PDFの内容（質問文など）はリポジトリにそのまま公開しないよう注意する
- 規準値の単位（項目平均 vs 素点合計）は必ず確認する — 間違えると偏差値が狂う
- クラスタ条件は論文に明記された判別基準のみ使用する（独自解釈しない）
- 抽出が不確かな部分は必ずユーザーに確認してから進む
- 複数PDFで情報が分散している場合は、すべて読んでから統合する（途中で登録しない）
